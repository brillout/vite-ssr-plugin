The `vike-solid` package is a <Link href="/extensions">Vike extension</Link> that integrates [Solid](https://solidjs.com) in a full-fledged manner, providing a DX like a regular frontend framework such as Solid Start.

Source code: [GitHub > `vikejs/vike-solid`](https://github.com/vikejs/vike-solid)  
Version history: [`MIGRATION.md`](https://github.com/vikejs/vike-solid/blob/main/MIGRATION.md)  
Examples: [Bati](https://batijs.dev) and [`examples/`](https://github.com/vikejs/vike-solid/tree/main/examples)  

The documentation for using `vike-solid` can be found throughout this website (`vike.dev`).

> The guides, such as <Link href="/data-fetching" />, assume you're using Vike with <UiFrameworkExtension name comma />. If you aren't then see the sections `Without vike-{react,vue,solid}` such as <Link href="/useData#without-vike-react-vue-solid" />.

You can also integrate Solid yourself instead of using `vike-solid`, see <Link href="/solid" />.


## Scaffold new `vike-solid` app

Use [Bati](https://batijs.dev) for creating new apps.

> If you select Solid then Bati scaffolds a Vike + `vike-solid` app.


## Add `vike-solid` to existing Vike app

To add `vike-solid` to an existing Vike app: install the `vike-solid` npm package (e.g. `$ npm install vike-solid`) then extend your existing `+config.js` file (or create one) with `vike-solid`:

```js
// /pages/+config.js

import vikeSolid from 'vike-solid/config' // [!code ++]

export default {
  // ...
  extends: [vikeSolid] // [!code ++]
}
```

You can then use <Link href="#under-the-hood">new settings introduced by `vike-solid`</Link>:

```js
// /pages/+config.js

import vikeSolid from 'vike-solid/config'

export default {
  // ...

  // Setting to toggle SSR // [!code ++]
  ssr: false, // [!code ++]

  extends: [vikeSolid]
}
```


## Under the hood

The `vike-solid` extension is only `542` lines of code ([as of Mai 2024](https://gist.github.com/brillout/e2134686d682e9a450e10e95f1ff570b)). Browsing the [source code of `vike-solid`](https://github.com/vikejs/vike-solid) is therefore very much an option for deepening your understanding of Vike or for debugging.

Basically `vike-solid` does this:
- Sets <Link href="/hooks">Vike hooks</Link> and <Link href="/settings">Vike settings</Link> on your behalf. (Most notably <Link href="/onRenderHtml">`onRenderHtml()`</Link> and <Link href="/onRenderClient">`onRenderClient()`</Link>, see their implementation at
  [`renderer/onRenderHtml.tsx`](https://github.com/vikejs/vike-solid/blob/main/vike-solid/renderer/onRenderHtml.tsx)
  and
  [`renderer/onRenderClient.tsx`](https://github.com/vikejs/vike-solid/blob/main/vike-solid/renderer/onRenderClient.tsx).)
- Introduces new <Link href="/meta">custom hooks</Link> and <Link href="/meta">custom settings</Link> (e.g. <Link href="/ssr">`ssr`</Link> and <Link href="/stream">`stream`</Link>).
- Introduces new component hooks (e.g. <Link href="/useData">`useData()`</Link> and <Link href="/usePageContext">`usePageContext()`</Link>).

It implements following features:

- <Link noBreadcrumb href="/usePageContext" />
- <Link noBreadcrumb href="/Layout" />
