### 高速なHMR

とても高速なHMR。powered by Vite <ViteLogo />

HMRは、Viteの斬新なlazy-transpilingのおかげで、スケールを拡大しても高速を維持します。

### ルーティング

ページは自動的にルーティングされます：

```
ファイル                      URL
pages/index/+Page.js         /
pages/about/+Page.js         /about
pages/movie/@id/+Page.js     /movie/1, /movie/2, /movie/3, ...
```

いわゆる _Route Strings_ や _Route Functions_ も使えます。

```js
// Route String
export default '/movie/@id'

// Route Function — より柔軟
export default pageContext => pageContext.urlPathname === '/admin' && pageContext.isAdmin
```

どちらかを選択可能：

- より高速なページナビゲーションのためのクライアントルーティング。（ページナビゲーション時にDOMの変異が発生します。）
- よりシンプルなアプリのアーキテクチャのためのサーバールーティング。（ページナビゲーション時にHTMLをフルリロード）

### レンダリングモード (SSR, SPA, SSG, HTML-only)

各ページで、以下のいずれかを選択できます：

- SSR: ページのHTMLはリクエスト時にレンダリングされます。
- SSG: ページのHTMLはビルド時にプリレンダリングされます。
- SPA: ページはHTMLにレンダリングされません。ブラウザで読み込まれ、レンダリングされるだけです。
- HTML-only: ページはHTMLのみにレンダリングされ、ブラウザサイドのJavaScriptはゼロまたは最小限になります。

例えば、管理画面をSPAとしてレンダリングする一方で、マーケティングページをHTMLのみにレンダリングし、ブラウザ側のJavaScriptをゼロにすることができます。

このウェブサイト`https://vike.dev`はHTMLのみで、数KBのJavaScript（このページの折りたたみ可能なLEARN MOREセクションなど、わずかなインタラクティブ機能を実装している）しか読み込んでいません。
