You can use Vike with any backend:
- Java (Spring, Grails, ...)
- PHP (Laravel, Symfony, CakePHP, ...)
- Ruby on Rails
- Python (Django, Flask, FastAPI, ...)
- Elixir (Phoenix, ...)
- Go (Gin, Echo, Fiber, ...)
- Rust (Actix Web, Rocket, ...)
- Backend-as-a-Service (Firbase, ..)
- ...

## Common setup

The usual setup is:
 - Using your backend to implement the business logic of data, and Vike to implement rich user interfaces.
 - <Link href="/pre-rendering">Pre-render (SSG) your Vike app</Link> and deploy it to a static host (<Link href="/github-pages" noBreadcrumb />, <Link href="/cloudflare-pages" noBreadcrumb />, <Link href="/netlify" noBreadcrumb />, <Link href="/static-hosts">etc</Link>).
